# synthonyx-kit-primitives

Parameter-source macros and const-generic getter types. Every type
generated by these macros implements
[`synthonyx_kit_core::Get<T>`](../synthonyx-kit-core).

## What's here

- `param!(Name: Type = expr)` — embed a value as a type.
- `env_param!(Name = "VAR")` / `env_param!(Name = "VAR" or "default")` —
  read an environment variable at call time.
- `ConstI8..ConstI128`, `ConstIsize`, `ConstU8..ConstU128`, `ConstUsize`,
  `ConstBool` — pass numeric / bool literals via const generics.

```rust
use synthonyx_kit_primitives::{param, ConstU32};
use synthonyx_kit_core::Get;

param!(Greeting: String = "Hello, World!");
assert_eq!(Greeting::get(), "Hello, World!");
assert_eq!(ConstU32::<42>::get(), 42);
```

The pattern is to declare a zero-sized type whose `Get::<T>::get()` returns
the desired value. Anywhere an RTM's `Config` extension trait names a
`Get<T>` associated type, you supply a type defined with one of these
macros instead of a runtime value.
